📝 Pașii de Setup pentru Restaurant Nou

0

Pregătire (înainte de a începe)

Pregătește în Excel: lista ingredientelor, meniul actual, rețetele (fișe tehnice), și stocul curent.

1

Introducerea Ingredientelor

Adaugă toate materiile prime: nume (RO/EN), categorie, unitate, preț, furnizor, waste %, alergeni.

2

📋 Adaugă Produsul în Catalog

Unde: Admin v4 → Administrare → Catalog Produse → ➕ Adaugă Produs

Ce completezi:

  • Nume (RO + EN): Ex: "Pizza Margherita" / "Margherita Pizza"
  • Categorie: Ex: Pizza, Paste, Ciorbe, Băuturi
  • Preț de vânzare: Ex: 35 RON (estimativ inițial)
  • Descriere (RO + EN): Descriere apetisantă a produsului
  • Alergeni: Selectează din listă (Lactate, Gluten, etc.)
  • Info suplimentare: Gramaj, timp preparare, nivel picanterie
  • Imagine: URL sau upload (opțional)

⚠️ IMPORTANT: După salvare, notează ID-ul produsului (ex: 343) - vei avea nevoie pentru rețetă!

✅ Produsul va apărea IMEDIAT în meniul clientului!

3

📝 Creează Rețeta (pentru gestionare stoc)

Unde: Admin v4 → Stocuri → Rețete

Pași:

  1. Click "Selectează Produs" → Alege produsul nou (ex: ID 343)
  2. Click "➕ Adaugă Ingredient"
  3. Selectează ingredientul din listă (ex: Mozzarella)
  4. Introdu cantitatea necesară (ex: 200g)
  5. Repetă pentru toate ingredientele
  6. Click "💾 Salvează Rețeta"

Rezultat: Se creează automat server/retete/retete-actuale/produs-343-pizza-margherita.json

⚠️ CRUCIAL: Fără rețetă, stocurile NU se scad automat la vânzare!

4

💰 Verifică Fișa Tehnică (optimizare preț)

Unde: Admin v4 → Stocuri → Fișe Tehnice

Ce se întâmplă:

  1. Selectează produsul tău (ID 343)
  2. Sistemul calculează AUTOMAT din rețetă:
    • Cost ingrediente (ex: 12.50 RON)
    • Preț de vânzare (ex: 35 RON - setat la Pasul 2)
    • Marja de profit (ex: 22.50 RON = 64.3%)
  3. Dacă marja e prea mică/mare → Ajustează prețul în Catalog Produse

Exemplu:

Pizza Margherita
━━━━━━━━━━━━━━━━━━━━━━━━
• Mozzarella (200g): 9.00 RON
• Sos roșii (150ml): 2.25 RON  
• Aluat (300g): 1.25 RON
━━━━━━━━━━━━━━━━━━━━━━━━
TOTAL COST: 12.50 RON
Preț vânzare: 35.00 RON
Marja profit: 22.50 RON (64.3%)
5

✅ Rezultat Final

După finalizarea pașilor:

  • ✅ Produsul apare în meniul clientului (Pasul 2)
  • ✅ Stocurile se scad automat la vânzare (Pasul 3)
  • ✅ Costurile sunt calculate corect (Pasul 4)
  • ✅ Prețul este optimizat pentru profit maxim

💡 VARIANTĂ RAPIDĂ: Dacă vrei doar să apară produsul în meniu (fără gestionare stocuri), completează doar Pasul 2! Rețeta și Fișa Tehnică le poți face mai târziu.

6

Testare Completă

Testează o comandă end-to-end: plasare → KDS → livrare → plată → bon nefiscal.

💡 Tips & Best Practices

✅ Folosește Unități Consistente

Alege kg, L, buc pentru toate ingredientele. Evită conversiile multiple (g → kg) care pot genera erori de rotunjire.

📊 Monitorizează COGS

Verifică costul ingredientelor (COGS) în Dashboard Advanced. Target: 28-35% din preț pentru marjă profitabilă.

🔄 Actualizează Stocurile

Folosește NIR (Notă Intrare-Recepție) pentru fiecare livrare de ingrediente. Sistemul actualizează automat stocurile și prețul mediu ponderat (WAC).

🗑️ Waste Tracking

Definește procentajul de pierdere pentru fiecare ingredient (ex: legume 10%, carne 5%). Sistemul calculează automat COGS real.

🎯 Gata să Începi?

Fie că alegi să începi cu rețetele existente sau să creezi propriul meniu, aplicația este gata să te sprijine în fiecare pas!

🔧 Troubleshooting

🚫 Serverul nu Pornește

Symptome:

  • Eroare "Port 3000 already in use"
  • Serverul se oprește imediat

Soluții:

# Verifică procesele pe portul 3000
netstat -an | findstr :3000

# Oprește toate procesele Node.js
taskkill /F /IM node.exe

# Repornește serverul
npm start

🔌 Erori de Conectivitate la ANAF

Symptome:

  • "Autentificarea la ANAF a eșuat"
  • "Timeout la conectare"
  • "Certificat invalid"

Soluții:

  1. Verifică certificatul digital în fiscal-config.properties
  2. Testează conectivitatea: ping api.anaf.ro
  3. Verifică validitatea certificatului

💾 Probleme cu Backup-ul

Symptome:

  • "Backup-ul fiscal a eșuat"
  • "Spațiu insuficient pe disk"
  • "Permisiuni insuficiente"

Soluții:

# Verifică spațiul pe disk
dir C:\

# Verifică permisiunile
icacls C:\fiscal_bk

# Rulează ca Administrator
.\scripts\setup-backup.ps1

📝 Erori de Validare XML

Symptome:

  • "XML invalid"
  • "Schema XSD nu se potrivește"
  • "Semnătură digitală invalidă"

Soluții:

  1. Navighează la "Verificare Semnătură Digitală"
  2. Upload un fișier XML de test
  3. Verifică rezultatele
